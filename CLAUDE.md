# Infra リポジトリ運用ルール

## このリポジトリについて

AWS/Azure/Google Cloud/AIに関する個人の学習・実務メモを管理するリポジトリ。共同編集者はおらず、CI/CDも組んでいない個人用リポジトリ。

## Git運用（自動実行してよい範囲）

- ドキュメントの追加・修正を行い、内容が完成した区切りで、都度 `git add <対象ファイル>` → `git commit` してよい。ユーザーに都度確認する必要はない。
- コミット後は都度 `git push origin main` してよい。個人用リポジトリで他者への影響がないため、プッシュ前の確認も不要。
- ただし以下は事前確認する。
  - `git add -A` / `git add .` など対象を絞らない一括ステージ（意図しないファイル混入を避けるため、変更したファイルを明示的に指定する）
  - force push、履歴改変（rebase -i、reset --hardなど）
  - `.gitignore` に該当しそうな秘匿情報を含むファイルの追加
- コミットメッセージは変更内容が分かる一言で簡潔に書く。
- 詳細なコマンド手順は [common/git-workflow.md](common/git-workflow.md) を参照。

## ドキュメント編集の方針

- 既存ファイルの構成・見出しスタイル（概要/主要トピック/設計ポイント/運用ポイント/ベストプラクティス等）に合わせる。
- 新しいトピックを追加した場合は、該当フォルダの `README.md` の一覧表にもリンクを追加する。
