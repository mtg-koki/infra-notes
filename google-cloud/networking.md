# Google Cloud ネットワーキング

## 概要

Google Cloudのグローバルネットワークを活かしたVPC設計。VPCがグローバルリソースである点（リージョンをまたぐサブネット構成）がAWS/Azureとの大きな違いとなる。

## 主要サービス

- VPC（グローバルリソース、リージョンごとのサブネット）
- Cloud Interconnect / Cloud VPN（オンプレ接続）
- Cloud Load Balancing（グローバル/リージョナル、L4/L7）
- Private Service Connect（PaaSへのプライベート接続）
- Cloud DNS

## 設計ポイント

- VPCはグローバルリソースのため、リージョン間のサブネット追加が容易。将来のマルチリージョン展開を見越した設計がしやすい。
- 共有VPC（Shared VPC）でホストプロジェクトにネットワークを集約し、サービスプロジェクトから利用する構成を検討する。
- グローバルロードバランサーを使うことで、単一のエニーキャストIPで複数リージョンへのトラフィック分散が可能。

## 運用ポイント

- VPC Flow Logsで通信を可視化し、異常検知に活用する。
- Cloud Interconnectの冗長構成（複数ロケーション、複数プロバイダ）を検討する。
- ファイアウォールルールの階層（組織レベル/VPCレベル）を整理し、優先度の衝突を避ける。
