# AWS IAM

## 概要

AWSにおけるアクセス制御の中核サービス。IAMユーザー/ロール/ポリシーに加え、Organizations・IAM Identity Centerによるマルチアカウント統制までを含めて扱う。

## 主要サービス

- IAM（ユーザー、グループ、ロール、ポリシー）
- AWS Organizations（マルチアカウント管理、SCP）
- IAM Identity Center（旧SSO、フェデレーション）
- AWS STS（一時的な認証情報）

## 設計ポイント

- 人間のアクセスはIAM Identity Center経由のフェデレーションに統一し、IAMユーザーの直接作成は避ける。
- ワークロードへの権限付与はIAMロール（EC2インスタンスプロファイル、Lambda実行ロール等）を用い、アクセスキーの埋め込みを避ける。
- SCPで組織全体のガードレール（禁止リージョン、禁止アクションなど）を設定し、各アカウントのIAMポリシーは業務要件に絞る。

## 運用ポイント

- IAM Access Analyzerで未使用の権限・外部共有リソースを定期的に検出する。
- 特権操作（ルートユーザー利用、IAMポリシー変更等）はCloudTrailで監視し、アラート対象にする。
- アクセスキーのローテーションと、長期未使用の認証情報の自動無効化を運用に組み込む。
