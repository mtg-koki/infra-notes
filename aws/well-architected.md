# AWS Well-Architected Framework

## 概要

AWSが提唱する設計・運用のベストプラクティス集。6つの柱（Pillar）で構成され、アーキテクチャレビューの標準的な評価軸として使う。

## 主要サービス

- 6つの柱: 運用上の優秀性 / セキュリティ / 信頼性 / パフォーマンス効率 / コスト最適化 / 持続可能性
- AWS Well-Architected Tool（レビューの実施・記録）
- Trusted Advisor（自動チェックによる推奨事項）

## 設計ポイント

- 各柱はトレードオフの関係にある（例: 信頼性向上とコスト最適化は相反しうる）。要件の優先順位を明確にした上でバランスを取る。
- 新規設計時だけでなく、既存システムへのレビュー適用（Well-Architectedレビュー）にも活用する。

## 運用ポイント

- 定期的なレビュー（四半期〜半年に一度など）を実施し、High Riskの指摘事項に対応計画を立てる。
- Trusted Advisorの推奨事項を運用のチェックリストに組み込む。

## ベストプラクティス

- レビュー結果は放置せず、改善バックログとして継続的にトラッキングする。
