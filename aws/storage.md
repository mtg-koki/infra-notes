# AWS ストレージ

## 概要

オブジェクトストレージ（S3）、ブロックストレージ（EBS）、ファイルストレージ（EFS）など、用途別のストレージサービスとライフサイクル管理の観点をまとめる。

## 主要サービス

- S3（ストレージクラス、ライフサイクルポリシー、バージョニング）
- EBS（ボリュームタイプ、スナップショット）
- EFS（マネージドNFS、マルチAZアクセス）
- Storage Gateway（オンプレとのハイブリッドストレージ）
- Backup（バックアップの一元管理）

## 設計ポイント

- S3はアクセス頻度に応じたストレージクラス（Standard/IA/Glacier等）とライフサイクルルールでコストを最適化する。
- EBSはIOPS/スループット要件に応じてボリュームタイプ（gp3/io2等）を選定する。
- 複数インスタンスからの同時アクセスが必要な場合はEFS、単一インスタンス専有ならEBSを選ぶ。

## 運用ポイント

- S3バケットポリシー・パブリックアクセスブロック設定を組織全体で統制する（誤公開の防止）。
- バックアップはAWS Backupで一元化し、リストア試験を定期的に実施する。
- スナップショットの世代管理・保持期間ポリシーを定義し、コスト増大を防ぐ。
