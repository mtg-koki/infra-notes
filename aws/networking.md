# AWS ネットワーキング

## 概要

VPCを中心としたAWSのネットワーク設計。マルチアカウント/マルチVPC環境での接続方式（Transit Gateway、PrivateLink）とグローバル配信（CloudFront、Route 53）を含めて扱う。

## 主要サービス

- VPC（サブネット、ルートテーブル、NACL、セキュリティグループ）
- Transit Gateway（マルチVPC/オンプレ接続のハブ）
- PrivateLink / VPCエンドポイント（AWSサービスへのプライベート接続）
- Route 53（DNS、ヘルスチェック、ルーティングポリシー）
- CloudFront（CDN）、Direct Connect（専用線）

## 設計ポイント

- マルチアカウント環境では、Transit Gatewayによる集約型接続かVPCピアリングのメッシュ型かをアカウント数・トラフィックパターンで選定する。
- S3やDynamoDBなどへのアクセスはVPCエンドポイント経由にし、インターネット経由の通信を避ける。
- セキュリティグループはステートフル、NACLはステートレスという違いを踏まえ、多層防御として併用する。

## 運用ポイント

- VPC Flow Logsを有効化し、異常な通信パターンを検知できるようにする。
- Route 53のヘルスチェックとフェイルオーバールーティングで、リージョン障害時の切り替えを自動化する。
- IPアドレス枯渇を避けるため、VPC CIDRとサブネット設計は拡張性を考慮する（IPAMの活用）。
