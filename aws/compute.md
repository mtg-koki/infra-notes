# AWS コンピュート

## 概要

EC2を基盤とした仮想マシンから、コンテナ（ECS/EKS）、サーバーレス（Lambda）まで、ワークロード特性に応じたコンピュートサービスの選定観点をまとめる。

## 主要サービス

- EC2（インスタンスタイプ、購入オプション: On-Demand/Reserved/Savings Plans/Spot）
- ECS / Fargate（コンテナオーケストレーション、サーバーレスコンテナ）
- EKS（マネージドKubernetes）
- Lambda（サーバーレス、イベント駆動実行）
- Auto Scaling（水平スケーリング）

## 設計ポイント

- ワークロードの特性（常時稼働 / バースト / イベント駆動）でEC2・ECS/Fargate・Lambdaを使い分ける。
- ステートレス設計を前提にAuto Scalingを組み込み、単一障害点を排除する。
- コンテナ基盤はEKS（Kubernetes標準に寄せたい/マルチクラウド前提）かECS（AWSネイティブでシンプルに運用したい）かをチーム習熟度と要件で選ぶ。

## 運用ポイント

- Spotインスタンスは中断を前提とした設計（チェックポイント、耐障害性）で組み込み、コスト最適化に活用する。
- Lambdaはコールドスタート、同時実行数上限、タイムアウトの制約を踏まえて設計する。
- パッチ適用はSystems Manager Patch Managerで自動化する。
