# Azure Virtual Desktop (AVD)

## 概要

Azure上でWindows仮想デスクトップ環境を提供するマネージドDaaS（Desktop as a Service）。ホストプール・アプリグループ・ワークスペースの構成単位を理解し、多人数利用時のスケーラビリティとコストのバランスを取ることが要点となる。

## 主要サービス

- ホストプール（Personal / Pooled、負荷分散方式）
- セッションホスト（VMサイジング、FSLogix によるユーザープロファイル管理）
- アプリグループ（RemoteApp / デスクトップ配信）
- ワークスペース（ユーザーへのフィード提供）
- FSLogix Profile Container（プロファイルのAzure Files/NetApp Files配置）

## 設計ポイント

- Personal（1ユーザー1VM専有）かPooled（複数ユーザーでVM共有）かをユーザー要件（カスタマイズ性 vs コスト効率）で選定する。
- FSLogixプロファイルの格納先（Azure Files Premium / Azure NetApp Files）はIOPS要件とユーザー数で選ぶ。
- セッションホストのVMサイズとユーザー密度（Pooledの場合）は、想定アプリケーションの負荷特性に基づきサイジングする。

## 運用ポイント

- スケーリングプラン（Scaling Plan）で営業時間外のセッションホストを自動停止し、コストを最適化する。
- FSLogixのプロファイル肥大化・破損はログイン遅延の主要因になるため、監視とクリーンアップ運用を組み込む。
- AVD Insightsで接続品質・ログオン時間・ホストの負荷を監視する。

## ベストプラクティス

- ゴールデンイメージ（VMイメージ）の更新パイプラインを整備し、アプリ更新・パッチ適用を自動化する。
