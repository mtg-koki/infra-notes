# AI（生成AI活用 / Claude）

## 概要

Claudeを中心とした生成AIの実務活用に関する整理。プロンプト設計、外部システム連携（MCP）、組織導入時のガバナンスまでを実務観点でまとめる。クラウドインフラ領域とは異なり技術の変化が速いため、内容の鮮度（確認日）を明記して管理する。

## 設計ポイント

- 生成AIをワークフローに組み込む際は「人間のレビューが必須の工程」と「自動化してよい工程」を明確に切り分ける。
- モデル・APIのバージョンアップにより挙動が変わり得るため、重要なワークフローはプロンプト・出力のテストを整備する。
- 機密情報の取り扱い（学習への利用有無、データ保持ポリシー）を利用規約ベースで確認してから業務データを投入する。

## 運用ポイント

- API利用のコスト（トークン課金）を監視し、想定外のコスト増を検知する。
- プロンプト・システムプロンプトはコードと同様にバージョン管理する。
- モデルの出力品質は定期的に評価し、劣化・変化を検知する仕組みを持つ。

## ベストプラクティス

- 一次情報（公式ドキュメント）で確認できない仕様は「要検証」と明記し、推測で断定しない。
